# images2md.py ユーザーマニュアル ![License: MIT](https://img.shields.io/badge/License-MIT-blue.svg)
 [🇺🇸 English](README.md) or [🇯🇵 日本語](readme-ja.md) 
 
`images2md.py` は、フォルダ内にある複数の画像（スライドやスクリーンショットなど）を、1つの「Markdown（マークダウン）ファイル」へ自動でまとめてくれる便利なツールです。

外部のインターネットサービスと通信しないため、機密情報が含まれる画像でも安全に使用できます。また、元の画像ファイルを書き換えたり消したりすることもありません。

## 🛠️ 動作に必要な環境
- **Python 3.10 以上** がインストールされていること。
- 追加のパッケージ（外部ライブラリ）のインストールは一切不要です。

> 💡Macでは[Python公式サイト](https://www.python.org/downloads/macos/)からインストーラーをダウンロードするか、「ターミナル」で `python3` と入力してインストールが可能です。Windows11ではMicrosoft Storeから直接インストール可能です。


---

## 🚀 基本的な使い方

ツールの実行は、コマンド入力画面から行います。
例えばMacの場合は「**ターミナル**」、Windowsの場合は「**PowerShell**」を開くことができます。

### 1. コマンドの入力
`slides` フォルダの中にある画像を読み込んで、`presentation.md` という名前のMarkdownファイルを作成する場合の例です。images2md.pyはtoolsフォルダに入っている前提です。以下のコマンドをコピーして画面に貼り付け、Enterキーを押して実行します。


```bash
python tools/images2md.py ./slides -o presentation.md

```
```
📂 現在のフォルダ
│
├── 📂 tools
│   └── 📄 images2md.py      ←─ スクリプトファイル
│
├── 📂 slides                ←─ スライドがあるフォルダ (./slides)
│   ├── 🖼️ slide1.png
│   ├── 🖼️ slide2.png
│   └── 🖼️ slide10.png
│　　└── ...
```



### 2. 実行結果の確認

処理が成功すると、指定したフォルダに「presentation.md」のファイル名のMarkdownファイルが作成されます。

* 💡 **ポイント:** 初期設定では、画像がBase64形式というデータに変換され、**Markdownファイルの中に直接埋め込まれます**。画像が1つのファイルにまとまるため、どのような環境でも表示されるので便利です。

## ⚙️ 便利なオプション機能

コマンドの末尾に付け足すことで、動作を自由にカスタマイズできます。

| オプション | 意味・使い方 | 初期値 |
| :--- | :--- | :--- |
| `-o ファイル名`<br>`--output ファイル名` | 出力するMarkdownのファイル名を指定します。 | `presentation.md` |
| `--link` | 画像を埋め込まず、画像への「相対パスのリンク」として書き出します（ファイルサイズを軽くしたい時用）。 | 無効（埋め込み） |
| `--max-kb 数値` | 処理する画像の最大サイズ（キロバイト）を指定します。これを超える画像は自動でスキップされます。 | `300` (KB) 💡|
| `--heading-level 数値` | 各画像の上に付く見出しの大きさを `1`〜`6` で指定します（例: `3` を指定すると `### 見出し` になります）。 | `2` (`## 見出し`) |
| `--title "タイトル"` | Markdownの最上部に、文書全体のタイトルを追加します。 | なし |
| `--overwrite` | すでに同じ名前のMarkdownファイルが存在する場合、強制的に上書きします。 | 無効（エラー終了） |
| `--dry-run` | 実際にはファイルを書き出さず、どの画像が処理され、どの画像がスキップされるかの予定だけ画面に表示します。 | 無効 |
| `--version` | ツールのバージョンを表示します。 | - |
| `-h`, `--help` | ヘルプ（使い方の説明）画面を表示します。 | - |

>💡 通常、PowerPointから書き出したスライド画像は300KB以下です。もし「画像がスキップされた（終了コード2）」場合は、PowerPointの画質設定を確認するか、--max-kb 500 などで上限を引き上げてみてください。
> ⚠️ **注意：** `--embed`（埋め込みモード・初期値）と `--link`（リンクモード）を同時に指定することはできません。

## 📂 対応している画像と並び順

### 対応しているファイル形式

拡張子（大文字・小文字は問いません）が以下の画像のみ処理します。

* `.png`
* `.jpg` / `.jpeg`
* `.webp`
※PDF、GIF、動画、フォルダなどは自動的にスキップされます。

### ⚠️ 画像を探す範囲

指定したフォルダの**直下（1階層目）にある画像だけ**を探します。フォルダの中にある「別のフォルダ」の中身までは見に行きません。

### 🔢 画像の見出しと並び順

ファイル名は自動的に「自然な順序（Natural Order）」で並び替えられます。

* 例：`Slide1.png` ➔ `Slide2.png` ➔ `Slide10.png` の順に正しく並びます。

また、作成されるMarkdownの見出しには、画像ファイル名から拡張子を取り除いた名前が自動で使われます。
（例：`Slide1.png` の場合 ➔ `## Slide1`）

---

## 🚦 終了コード（エラーの判断）

システムやスクリプトからこのツールを呼び出す場合、以下の終了コードで成否を判定できます。

* **`0`**: 成功（すべての画像が正常に処理されました）
* **`1`**: 致命的なエラー（入力フォルダが見つからない、引数の間違い、書き込み失敗など）
* **`2`**: 処理は完了しましたが、サイズ超過や非対応などの理由で**スキップされた画像が1つ以上あります**。
